# Causes of Latency Variation

## 1. Retrieved Context Size

Different queries retrieve different amounts of information from the vector database.

Example:

* Query: "What is FAISS?" → Small context
* Query: "Explain Hybrid Search and compare it with Vector Search" → Larger context

Larger retrieved context increases prompt size and requires additional token processing.

### Impact

* Smaller context → Lower latency
* Larger context → Higher latency

---

## 2. Output Length

Different questions require different answer lengths.

Example:

* "What is BM25?" → Short answer
* "Compare BM25, Vector Search and Hybrid Search" → Longer answer

Since LLMs generate tokens sequentially, longer outputs increase generation time.

### Impact

* Short output → Faster generation
* Long output → Slower generation

---

## 3. Query Complexity

Simple factual questions require minimal reasoning.

Example:

```text
What is RAG?
```

Complex questions require comparison, analysis, and multi-step reasoning.

Example:

```text
Why does Hybrid Search outperform pure Vector Search?
```

### Impact

* Simple reasoning → Lower latency
* Complex reasoning → Higher latency

---

## 4. Retrieval Quality

Latency is also affected by the quality of retrieved documents.

When highly relevant chunks are retrieved:

```text
Relevant Context
       ↓
Direct Answer
```

The model responds efficiently.

When retrieval quality is poor:

```text
Weak Context
      ↓
Additional Reasoning
```

The model spends more effort generating a coherent response.

### Impact

* High-quality retrieval → Faster response
* Poor retrieval → Slower response

---

## 5. Token Processing Requirements

LLMs process tokens rather than documents.

Each query introduces:

* User prompt tokens
* Retrieved context tokens
* Generated output tokens

Example:

```text
Query A
Input Tokens = 100
Output Tokens = 50

Query B
Input Tokens = 1000
Output Tokens = 400
```

Query B naturally requires more computation.

### Impact

More tokens result in increased latency.

# Analysis Using RAG Evaluation Metrics

## BLEU Score

BLEU measures overlap between the generated answer and a reference answer.

### Relationship to Latency

* Queries with better retrieval generally produce higher BLEU scores.
* Better retrieval reduces reasoning effort.
* Higher BLEU often indicates more efficient grounding.

### Observation

High BLEU scores usually correspond to queries where relevant context was retrieved successfully.

---

## ROUGE Score

ROUGE measures content coverage and recall.

### Relationship to Latency

* Longer and more detailed answers generally improve ROUGE scores.
* Longer responses require more token generation.
* Higher ROUGE often increases generation latency.

### Observation

Queries requiring comprehensive explanations tend to have both higher ROUGE scores and higher response times.

---

## Faithfulness

Faithfulness measures whether the generated answer is supported by retrieved documents.

### Relationship to Latency

When retrieved context is highly relevant:

* Faithfulness increases
* Hallucinations decrease
* Generation becomes more efficient

When retrieval quality decreases:

* Faithfulness decreases
* Additional reasoning is required
* Latency increases

### Observation

Faithfulness is strongly connected to retrieval quality and directly influences response efficiency.

# Latency vs Quality Trade-Off

| Factor              | Effect on Latency        | Effect on Quality         |
| ------------------- | ------------------------ | ------------------------- |
| Larger Context      | Higher                   | Better grounding          |
| Longer Output       | Higher                   | Better coverage           |
| Higher BLEU         | Neutral / Slightly Lower | Better accuracy           |
| Higher ROUGE        | Higher                   | Better completeness       |
| Higher Faithfulness | Lower hallucination risk | Better reliability        |
| Complex Reasoning   | Higher                   | Better analytical quality |

---

# Conclusion

The latency variation observed in Day 12 Colab 2 is primarily caused by differences in:

1. Retrieved context size
2. Prompt length
3. Output length
4. Query complexity
5. Retrieval quality
6. Total token count
7. End-to-end RAG pipeline processing

From an evaluation perspective:

* BLEU measures answer accuracy relative to references.
* ROUGE measures content coverage and completeness.
* Faithfulness measures grounding of responses in retrieved documents.

Queries that require larger context, longer outputs, and more reasoning naturally increase latency. However, these queries may also improve BLEU, ROUGE, and Faithfulness scores.

Therefore, latency variation is expected in production RAG systems and represents a trade-off between response speed and answer quality rather than a limitation of Groq infrastructure.
